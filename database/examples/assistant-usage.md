# Assistant Usage Examples

## 例 1：查询

用户：

```txt
帮我看看这周有哪些重要安排
```

建议流程：

1. secretary 判断这是查库请求
2. 调用 `list_records`
3. 时间范围设为本周
4. 读取结果后组织自然语言回复

---

## 例 2：语义搜索

用户：

```txt
帮我找最近和比赛报名有关的事情
```

建议流程：

1. secretary 理解“比赛报名”是语义条件，不是固定关键词
2. 调用 `search_records`
3. 返回真正相关的记录
4. 如有必要再调用 `get_record_detail`

---

## 例 3：新增事项

用户：

```txt
明天下午三点去图书馆和组员讨论报名材料
```

建议流程：

1. secretary 判断用户要录入
2. 调用 `create_drafts_from_text`
3. 返回待确认 draft
4. 由用户确认后再进入正式记录

---

## 例 4：普通聊天

用户：

```txt
你觉得我这周安排是不是太满了
```

建议流程：

1. secretary 先判断是否需要数据库
2. 如果需要，再查询本周安排
3. 如果不需要，也可以直接从上下文正常回答

重点：

不要因为进入了日历项目，就把所有输入都机械地变成数据库操作。
